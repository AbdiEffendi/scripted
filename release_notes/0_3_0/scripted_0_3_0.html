Scripted 0.3 - release notes

KDV
Added a Keybinding UI

It is now possible to redefine Scripted's keybindings via a simple UI. To access this UI open the 'Help panel' by clicking the question mark icon in the toolbar or pressing 'F1'. Click any of the shortcut keys to change it.

Any changes you make to the keybindings are saved in ${user.home}/.scriptedrc/keymap-${os.name}.json

There is no reset button yet. To reset the keybindings to their default values you can simply delete the keymap file.
Keybinding's related bugfixes

Scripted-260: Some keybindings/actions are now marked as Global and will work even if the editor does not currently have focus. E.g. 'Open file' dialog's keybinding.

Scripted-255: Opening help is now assigned to F1 by default.

Scripted-240: Fixed a bug that caused 'SHIFT-TAB' keybinding to occasionally get lost.
Enhancements to AMD configuration discovery

Recognize more idioms for extracting the AMD loader configuration from projects: Scripted-253, Scripted-239.

Scripted-252: Sloppy resolution disabled. Prior to 0.3 when a dependency was not being found a 'sloppy' resolution would kick in to look for any js file with a matching name. This would sometimes inadvertently resolve dependencies that should not be resolvable. This creates the impression that incorrect project structures are in fact correct. In 0.3 we have completely disabled this 'sloppy' mode and instead focus on making the resolution algos more accurate and complete.

Scripted-208: Minimal support for discovering AMD loader config in curl-based projects.

Scripted-238: Use AMD resolver config for AMD modules that are 'commonjs-wrapped'.

Scripted-237: Recognize 'packages' declarations in AMD loader configuration.
Enhancements to Open File and Look in Files search dialogs

Besides having a new look. These dialogs now perform a 'dynamic incremental' search. A search kicks of as soon as enough characters have been typed (0 characters for 'Open File' file-name searches and 3 characters for 'Look in File' text searches). As the search is still/already running you can continue to type and the still executing query is dynamically updated as you type. The backend optimizes dynamic query updates by using the already found results while changing the search-term for the remaing part of a still ongoing search.

: Searches are also automatically paused when the number of displayed results exceeds what can be visibly displayed on-screen. A paused-search is automatically resumed when you scroll near the end of the visible results.
Global Configuration via '.scritpedrc'
Parse config files using JSON5'
 All Scripted's json config files are now parsed using JSON5 instead of plain JSON. JSON5 is an extension of plain JSON JSON5 but has a more user-friendly, forgiving syntax. For example, it allows for comments in the config files.
